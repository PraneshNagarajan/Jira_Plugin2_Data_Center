Section 3.1.1 · Frontend Files

webpack.config.js — Full Annotated

 frontend/my-app/webpack.config.js
const WrmPlugin  = require('atlassian-webresource-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
const path = require('path');

module.exports = {

  // ENTRY — root file webpack starts bundling from
  // key 'app' is used in contextMap and becomes the bundle filename
  entry: { app: './src/App.tsx' },

  // OUTPUT — where built files land on disk
  // path: backend Maven resources folder — gets packaged into .jar
  output: {
    path: path.resolve(__dirname, '../backend/src/main/resources/frontend'),
    filename: 'js/bundled.[name].js',
    // publicPath: '' = relative paths; WRM resolves them at runtime
    // publicPath: '/' would write absolute paths → 404 (wrong for WRM)
    publicPath: '',
  },

  watch: true, // auto-rebuild on file changes (for atlas-run dev loop)
  resolve: { extensions: ['.tsx', '.ts', '.js'] },

  module: {
    rules: [
      { test: /\.tsx?$/, exclude: /node_modules/, use: 'ts-loader' },
      { test: /\.css$/,  use: [MiniCssExtractPlugin.loader, 'css-loader'] },
      { test: /\.(svg|png|jpg|jpeg|gif|ico)$/,
        type: 'asset/resource',
        generator: { filename: 'images/[name][ext]' } },
    ],
  },

  plugins: [
    new MiniCssExtractPlugin({ filename: 'css/[name].css', runtime: false }),
    new WrmPlugin({
      pluginKey: 'com.atlassian.aservo.myPlugin',
      // entry 'app' tagged with Jira context 'app'
      contextMap: { app: ['app'] },
      watch: true,
      // Must be inside META-INF/plugin-descriptors/ for OSGi scanner
      xmlDescriptors: path.resolve(__dirname,
        '../backend/src/main/resources/META-INF/plugin-descriptors/wr-defs.xml'),
    }),
  ],
};
LoaderFile typesOutput
ts-loader.tsx, .ts→ JavaScript
css-loader + MiniCss.css→ css/app.css
asset/resource.svg .png .jpg .gif .ico→ images/[name][ext]
🚫

publicPath must be '' (empty string)!
'/' writes src='/images/logo.png' → absolute path from web root → 404. WRM serves images at /jira/download/resources/{key}/images/ — these paths never match.
'' writes relative paths — WRM resolves them correctly at runtime.

💡

Best practice for images: Don't import images via webpack at all. Build the full WRM URL manually in React using AJS.contextPath() + PLUGIN_KEY + filename. See Slide 7 for the pattern.

⚠️

watch: true auto-rebuilds on file changes. Perfect for the atlas-run dev loop — Jira detects new files and hot-reloads automatically.

Section 3.1.4 · Root React Component

App.tsx — Root Component & React Mount

 frontend/my-app/src/App.tsx
import React from 'react'; // required with jsx: "react"
import ReactDOM from 'react-dom'; // DEFAULT import — NOT: import * as ReactDOM
import './App.css';
import HomePage from './Page/HomePage';

interface IProps { contextPath: string; }

// pluginKey : web-resource-key — must match wr-defs.xml <web-resource key='entrypoint-app'>
const PLUGIN_KEY = 'com.atlassian.aservo.myPlugin:entrypoint-app';

export default function App({ contextPath }: IProps) {

  // Build full Atlassian WRM URL for any static asset:
  // → /jira/download/resources/com.atlassian.aservo.myPlugin:entrypoint-app/images/logo.svg
  const resourceUrl = (name: string) =>
    `${contextPath}/download/resources/${PLUGIN_KEY}/${name}`;

  return (
    <div className='App'>
      <HomePage contextPath={contextPath} />
    </div>
  );
}

// MUST use 'load' event — Jira renders HTML via Velocity, not static index.html
// Without this: React runs before #container exists → React Error #299
window.addEventListener('load', () => {
  // AJS = Jira's global JS object, always present on any Jira page
  // AJS.contextPath() → '/jira' in standard install
  const contextPath = window.AJS ? window.AJS.contextPath() : '';
  const wrapper = document.getElementById('container');
  wrapper ? ReactDOM.render(<App contextPath={contextPath} />, wrapper) : false;
});
🚫

window.addEventListener('load') is REQUIRED!
Jira renders HTML via Velocity templates — not a static index.html. Without 'load', React runs before #container exists in the DOM → React Error #299: createRoot called on invalid container.

⚠️

Import ReactDOM as DEFAULT — not namespace:
import ReactDOM from 'react-dom' ✅
import * as ReactDOM from 'react-dom' ❌ — namespace object lacks .render()

🔗 resourceUrl() Pattern — How WRM URLs Work

Never import images via webpack (import logo from './logo.svg'). No publicPath value produces the correct WRM URL automatically. Instead, build the URL manually:

Image URL pattern
const PLUGIN_KEY = 'com.atlassian.aservo.myPlugin:entrypoint-app';
// contextPath = '/jira' (from AJS.contextPath())
const imgUrl = `${contextPath}/download/resources/${PLUGIN_KEY}/images/react.svg`;
// → /jira/download/resources/com.atlassian.aservo.myPlugin:entrypoint-app/images/react.svg

Section 3.2.3 – 3.2.7 · Backend Files

Servlet, wr-defs.xml, app.vm & Spring

MyPluginServlet.java
public class MyPluginServlet extends HttpServlet {

  @ComponentImport // Atlassian DI: inject from Jira's OSGi registry
  private final TemplateRenderer templateRenderer;

  @Inject // JSR-330 injection
  public MyPluginServlet(final TemplateRenderer templateRenderer) {
    this.templateRenderer = templateRenderer;
  }

  @Override
  protected void doGet(HttpServletRequest req, HttpServletResponse res)
      throws IOException {
    Map<String, Object> context = new HashMap<>();
    res.setContentType("text/html;charset=UTF-8");
    context.put("contextPath", req.getContextPath()); // '/jira'
    templateRenderer.render("templates/app.vm", context, res.getWriter());
  }
}

Section 5.1 – 5.2 · Common Issues

React Error #299 & Image 404s

🚨

React Error #299 — createRoot on invalid container
Root cause: React 18 uses createRoot() not compatible with Jira DC. Even with React 16: mounting synchronously runs before Jira renders #container.

✗ Wrong — React 18 pattern
import ReactDOM from 'react-dom/client';

ReactDOM
  .createRoot(document.getElementById('container'))
  .render(<App/>);
✓ Correct — React 16 + Jira
import ReactDOM from 'react-dom'; // default!

window.addEventListener('load', () => {
  const ctx = window.AJS?.contextPath() ?? '';
  const el = document.getElementById('container');
  el && ReactDOM.render(<App contextPath={ctx}/>, el);
});
🚨

Section 5.3 – 5.6 · Common Issues

TypeScript Errors, SVG Loader & Dependency Conflicts

🚨 Error: 'render' does not exist on typeof ReactDOM

Root cause: npm resolved @types/react-dom@^16 to v19+ (independent versioning). v18+ removed ReactDOM.render().

Fix
# Pin EXACT version — no ^ or ~
npm install @types/react@16.9.0 @types/react-dom@16.9.0 --save-dev --legacy-peer-deps

// Use default import (NOT namespace import):
import ReactDOM from 'react-dom';     // ✅ .render() on default export
import * as ReactDOM from 'react-dom'; // ❌ namespace lacks .render()
🚨 Error: "react-jsx" build failure / React is not defined

Root cause: "react-jsx" is the React 17+ automatic JSX transform. React 16 does not support it.

tsconfig.json fix
"jsx": "react"     // ✅ correct for React 16
"jsx": "react-jsx" // ❌ React 17+ only — WRONG
🚨 Error: Module parse failed — SVG Loader Error

Root cause: import logo from './react.svg' treats SVG as a JS module without a loader.

Recommended fix: Delete the import entirely. Use full WRM URL in React instead (see Slide 11).

Alternative fix: Add type declaration:

src/declarations.d.ts
declare module '*.svg' { const content: string; export default content; }
// + webpack.config.js must include the asset/resource rule for .svg
🚨 Error: ERESOLVE dependency conflict on npm install

Root cause: @testing-library/react@16 requires react@^18 but project has react@16.

Fix — install v12 (last version for React 16)
npm install @testing-library/react@12 --save-dev --legacy-peer-deps
# v12 is the last version compatible with React 16
